# CC Switch Next v3.16.5

## 亮点

- **默认 Sonnet 模型更新为 Claude Sonnet 5** — Anthropic、OpenRouter 和 AWS Bedrock 供应商预设的默认模型已切换至最新的 Claude Sonnet 5（100 万上下文窗口，128K 最大输出）
- **omp 稳定性改进** — 修复了多个与 omp（Oh My Pi）相关的 bug，涵盖供应商配置、MCP 同步及测试覆盖等方面

## 供应商预设

- Anthropic 直连 API 供应商默认模型改为 `claude-sonnet-5`
- OpenRouter 在所有受支持应用中的预设已更新为 `anthropic/claude-sonnet-5`
- AWS Bedrock 预设已更新为 `global.anthropic.claude-sonnet-5`
- 合并 PR [#6](https://github.com/benzhoupo/cc-switch-next/pull/6)

## Bug 修复

- 修复多个测试文件中 `McpApps` 缺少 `omp` 字段的问题
- 修复 `useDirectorySettings` 测试中 omp 目录配置 mock 错误
- 移除导入导出同步测试中残留的 `openclaw` 字段引用
- 禁用 Dependabot 自动依赖更新
- 启用 `main` 分支保护（`enforce_admins: true`）
